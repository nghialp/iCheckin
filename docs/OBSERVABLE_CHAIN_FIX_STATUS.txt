🔴 ISSUE REPORTED
════════════════════════════════════════════════════════════════════════════

"The link chain completed without emitting a value. This is likely 
unintentional and should be updated to emit a value before completing"

Location: src/graphql/client.ts - authLink

════════════════════════════════════════════════════════════════════════════

✅ ROOT CAUSE IDENTIFIED
════════════════════════════════════════════════════════════════════════════

Problem: Observable chain not explicitly handling next, error, and complete events

The authLink was passing the observer directly to subscribe() without:
  ❌ Explicitly emitting values through observer.next()
  ❌ Properly propagating errors through observer.error()
  ❌ Explicitly signaling completion through observer.complete()

When async errors occurred, they weren't being sent to the observer,
causing the Observable chain to complete without emitting a value.

════════════════════════════════════════════════════════════════════════════

🔧 FIX APPLIED
════════════════════════════════════════════════════════════════════════════

File: src/graphql/client.ts

Changed authLink Observable from:

❌ BEFORE:
─────────────────────────────────────────────────────────────────────────
const authLink = new ApolloLink((operation, forward) => {
  return new Observable(observer => {
    (async () => {
      try {
        // ... setup ...
        const subscription = forward(operation).subscribe(observer);
        return () => subscription.unsubscribe();
      } catch (e) {
        const subscription = forward(operation).subscribe(observer);
        return () => subscription.unsubscribe();
      }
    })();
  });
});

✅ AFTER:
─────────────────────────────────────────────────────────────────────────
const authLink = new ApolloLink((operation, forward) => {
  return new Observable(observer => {
    (async () => {
      try {
        // ... setup ...
        const subscription = forward(operation).subscribe({
          next: value => observer.next(value),        // ✅ Emit value
          error: err => observer.error(err),          // ✅ Propagate error
          complete: () => observer.complete(),        // ✅ Signal complete
        });
        return () => subscription.unsubscribe();
      } catch (e) {
        observer.error(e);                            // ✅ Handle error
      }
    })();
  });
});

🎯 OBSERVABLE CONTRACT EXPLAINED
════════════════════════════════════════════════════════════════════════════

Every Observable MUST call ONE of these terminal events:

  1. observer.next(value) - Emit a value to observer
  2. observer.error(err)  - Signal an error
  3. observer.complete()  - Signal completion

The old code wasn't explicitly calling these, which caused RxJS to warn
that the Observable chain completed without emitting a value.

🔗 PATTERN REFERENCE
════════════════════════════════════════════════════════════════════════════

This is the correct pattern for ALL Apollo Links creating Observables:

new ApolloLink((operation, forward) => {
  return new Observable(observer => {
    const subscription = forward(operation).subscribe({
      next: value => observer.next(value),      // ✅ Pass through value
      error: err => observer.error(err),        // ✅ Pass through error
      complete: () => observer.complete(),      // ✅ Signal completion
    });
    
    return () => subscription.unsubscribe();    // ✅ Cleanup
  });
});
