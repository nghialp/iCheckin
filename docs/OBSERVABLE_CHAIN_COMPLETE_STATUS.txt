CHANGE #1: Fix #1 - Explicit Observable Handlers (01/03/2026)
─────────────────────────────────────────────────────────────────────────

❌ BEFORE:
  const subscription = forward(operation).subscribe(observer);

✅ AFTER:
  const subscription = forward(operation).subscribe({
    next: value => observer.next(value),
    error: err => observer.error(err),
    complete: () => observer.complete(),
  });

CHANGE #2: Fix #2 - Proper Resource Cleanup (02/03/2026)
─────────────────────────────────────────────────────────────────────────

❌ BEFORE:
  return new Observable(observer => {
    (async () => {
      try {
        const subscription = forward(operation).subscribe({...});
        return () => subscription.unsubscribe();  // ❌ Wrong scope!
      } catch (e) {
        observer.error(e);
      }
    })();
  });

✅ AFTER:
  return new Observable(observer => {
    let subscription: any;  // ✅ Declare in correct scope
    
    (async () => {
      try {
        subscription = forward(operation).subscribe({...});
      } catch (e) {
        observer.error(e);
      }
    })();

    return () => {  // ✅ Return from correct scope
      if (subscription) {
        subscription.unsubscribe();
      }
    };
  });

Fix #2: Resource Cleanup Scope (02/03/2026)
─────────────────────────────────────────────────────────────────────────

Problem:
  • Cleanup function inside async IIFE never executed
  • Subscription not properly unsubscribed
  • Potential memory leaks

Solution:
  ✅ Moved subscription to outer scope: let subscription: any
  ✅ Moved cleanup to proper scope: return () => {...}
  ✅ Added safety check: if (subscription) {...}

Benefit:
  • Proper resource cleanup on Observable destruction
  • Memory leak prevention
  • Follows RxJS patterns correctly

🔍 OBSERVABLE CONTRACT
════════════════════════════════════════════════════════════════════════════

Every Observable MUST:

1. ✅ EMIT VALUES - through observer.next(value)
   └─ What it does: Sends data to subscribers

2. ✅ HANDLE ERRORS - through observer.error(err)
   └─ What it does: Propagates errors to error handlers

3. ✅ SIGNAL COMPLETION - through observer.complete()
   └─ What it does: Marks Observable as done

4. ✅ PROVIDE CLEANUP - return () => {...}
   └─ What it does: Cleans up resources when destroyed

Our implementation now follows ALL of these!

🎓 LESSONS LEARNED
════════════════════════════════════════════════════════════════════════════

1. Observable Scope Rules
   • Cleanup function must be returned from Observable constructor
   • Variables used in cleanup must be in correct scope
   • Async operations need careful scope management

2. RxJS Best Practices
   • Always use explicit Observable handlers
   • Don't pass observer directly to subscribe()
   • Handle all three event types: next, error, complete

3. Apollo Link Patterns
   • Follow the subscription/cleanup pattern
   • Handle errors at Observable level
   • Ensure proper error propagation

4. Memory Management
   • Always provide cleanup functions
   • Track subscriptions in correct scope
   • Test cleanup by monitoring resource usage
