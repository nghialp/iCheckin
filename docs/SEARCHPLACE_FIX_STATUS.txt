🔴 ISSUE REPORTED
════════════════════════════════════════════════════════════════════════════

Fragment "MAP_PLACE_FIELDS" cannot be spread here as objects of type "SearchPlace"
can never be of type "Place"

════════════════════════════════════════════════════════════════════════════

✅ ROOT CAUSE IDENTIFIED
════════════════════════════════════════════════════════════════════════════

GraphQL Schema has TWO DISTINCT TYPES for places:

  1. Place Type (for detail queries)
     ├─ id, name, address, types, rating
     ├─ thumbnail, mapboxId, lat, lng
     └─ trips

  2. SearchPlace Type (for search/nearby queries)
     ├─ name, address, types, rating
     ├─ mapboxId, lat, lng, thumbnail
     └─ distance ← Only in SearchPlace!

Problem: Queries using nearbyPlaces() and searchPlaces() return SearchPlace type,
but fragments were defined for Place type. GraphQL cannot spread Place fragments
on SearchPlace objects.

════════════════════════════════════════════════════════════════════════════

🔧 SOLUTION IMPLEMENTED
════════════════════════════════════════════════════════════════════════════

1. Created new fragment for SearchPlace type
   └─ Name: SEARCH_PLACE_FIELDS
   └─ Type: SearchPlace
   └─ Includes: distance field (available only in search results)

2. Updated all affected queries
   ├─ home.query.ts - GET_HOME_DATA
   ├─ map.query.ts - GET_NEARBY_PLACES
   └─ place.query.ts - SEARCH_PLACES

3. Updated fragment exports
   └─ fragments/index.ts - Added SEARCH_PLACE_FIELDS export

┌──────────────────────────────────────────────────────────────────┐

└──────────────────────────────────────────────────────────────────┘

Place Type (Schema):
┌──────────────────────────────────────────────────────────────────┐
│ type Place {                                                     │
│   id: ID!                                                        │
│   name: String!                                                  │
│   address: String                                                │
│   types: [String!]                                               │
│   rating: Float                                                  │
│   thumbnail: String                                              │
│   mapboxId: String                                               │
│   lat: Float!                                                    │
│   lng: Float!                                                    │
│   trips: [Trip!]          ← Only in Place                        │
│ }                                                                │
└──────────────────────────────────────────────────────────────────┘

Key Differences:
  • Place has: id, trips (NOT in SearchPlace)
  • SearchPlace has: distance (NOT in Place)
  • Both share: name, address, types, rating, thumbnail, mapboxId, lat, lng
